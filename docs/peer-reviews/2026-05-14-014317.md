# Peer Review: Core Refactor & Technical Debt Reduction

**Date:** 2026-05-14
**Reviewer:** Gemini CLI (Peer Reviewer Skill)
**Status:** ❌ REJECTED

## Executive Summary

The refactoring effort has successfully decomposed the monolithic `RecipeEditor` and `MealCalendarClient` components and introduced a standardized "Kitchen UI" library. However, the implementation has introduced **critical regressions** that have left the codebase in a broken state. Most notably, the `RecipeSaveData` type was deleted from `types/index.ts`, causing 49 TypeScript errors across the project. Additionally, the Playwright E2E suite is currently failing due to misconfigured environment variables and broken authentication flows.

## Detailed Findings

### 1. Critical Build Failure (Type Errors)

The deletion of `RecipeSaveData` from `types/index.ts` has caused widespread failures.

- **Files affected:** 13 files, including `RecipeEditor.tsx`, `actions.ts`, and multiple test files.
- **Impact:** The project cannot be built or deployed.
- **Recommendation:** Restore `RecipeSaveData` to `types/index.ts`. It is still the primary data structure for recipe creation and updates.

### 2. Broken `RecipeComponent` Refactor

The refactor of `RecipeComponent` into a discriminated union is a good architectural move, but it was not applied consistently across the codebase.

- **Issue:** Existing tests and components (e.g., `calculateMacros`, `RecipeView.test.tsx`) still use the old "flat" shape without the `type` property.
- **Issue:** `ingredientId` is now a required field in the `ingredient` variant of the union, but many existing usages omit it.
- **Recommendation:** Perform a surgical update of all usages to include the `type` property and ensure `ingredientId` is provided where necessary.

### 3. Playwright Configuration Error

`playwright.config.ts` was modified to include `env` blocks inside the `use` object.

- **Issue:** Playwright's `use` object does not support an `env` property. This causes the `MOCK_AUTH` environment variable to be ignored, breaking the E2E authentication flow.
- **Recommendation:** Move environment variables to the `webServer.env` block or use a `.env` file for the test runner.

### 4. Code Quality & Standards

- **Good:** The decomposition of `RecipeEditor` into `StepManager`, `IngredientSearch`, and `ComponentList` is well-executed and significantly improves readability.
- **Good:** The "Kitchen UI" components in `components/ui/` are clean and reusable.
- **Standard Violation:** The "Mandatory Regression Testing" rule was ignored. The developer proceeded with large-scale changes without resolving the resulting test failures.

## Edge Cases & Missing Scenarios

- **Empty States:** The new `ComponentList` and `StepManager` do not explicitly handle the "no items" state with a friendly UI message (though functionality seems to hold).
- **Circular Dependencies:** While tests exist for circular dependencies in `lib/recipes.ts`, the UI (`RecipeEditor`) does not yet provide proactive feedback if a user tries to add a recipe that would cause a cycle.

## Conclusion & Next Steps

The refactor is architecturally sound but execution was reckless. The implementation must be rolled back or fixed immediately to restore build integrity.

**Priority 1:** Restore `RecipeSaveData` and fix all `tsc` errors.
**Priority 2:** Fix `playwright.config.ts` and ensure `npm test` and `npx playwright test` pass 100%.
**Priority 3:** Complete the `RecipeComponent` migration in the test files.
