## 2. Code Structure, Efficiency & Readability

- **Macro Calculation Logic:** The recursive approach in `lib/recipes.ts` is elegant but has a data-loading flaw. It assumes the input `recipe` object is fully hydrated. In the current implementation of `app/recipes/[id]/page.tsx`, Prisma only fetches one level of components. Any recipe nested 2+ levels deep will report **0 calories** for the sub-components because their own components were not fetched.
- **Type Safety:** The use of `any` is prevalent throughout `lib/recipes.ts`, `RecipeEditor.tsx`, and `RecipePlayMode.tsx`. This bypasses TypeScript's benefits and increases the risk of runtime errors during complex recursive operations.
- **Component Design:** `RecipeEditor` is well-organized but growing large. Consideration should be given to breaking out `IngredientSearch` and `StepList` into sub-components.
- **Efficiency:** The fuzzy matching logic in `lib/ingredient-matcher.ts` is efficient and uses a sensible "significant words" approach to handle shorthand instructions.

## 3. Edge Cases & Missed Items

- **BUG-003 (Unresolved):** The bug tracking file identifies a need for standardized unit selection. The current `RecipeEditor` still uses a free-text input for units, which will lead to conversion failures if users type "ounces" instead of "oz".
- **Indirect Circular Dependencies:** `saveRecipe` prevents a recipe from containing itself (A -> A), but it does **not** detect indirect loops (A -> B -> A). This will cause a `RangeError: Maximum call stack size exceeded` in `calculateMacros`.
- **Timer Persistence:** Timers in `RecipePlayMode` are managed in local state. If a chef refreshes the page or their phone browser sleeps/reloads, all active timers are lost.
- **USDA Macro Bases:** The calculation logic assumes USDA macros are always per 100 units. While standard for many items, the system should ideally validate the `unitName` from the USDA response.
- **Sub-recipe UI:** As noted in Story 1, there is no way for a user to select "Tomato Sauce" (a saved recipe) as an ingredient for "Marinara" in the current Editor UI.
