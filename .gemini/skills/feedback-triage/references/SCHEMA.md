# Documentation Schemas

To maintain consistency for future agents, always use these structures when triaging feedback.

## BUGS.md Entry

```markdown
### BUG-XXX: Short Title

**Status:** Active
**Description:** Detailed explanation of the failure.
**Reproduction:** Step-by-step instructions to trigger the bug.
```

## UI/UX Polish Entry

```markdown
## [Section Name]

- [ ] **Feature Name:** Description of the tweak.
```

## Feature Epic (v1/v2)

```markdown
# Upcoming Stories: [Epic Name]

## Story: [User Story Title]

As a [role], I want [goal] so that [benefit].

### Acceptance Criteria

- [ ] AC 1
- [ ] AC 2

### Technical Notes

- Implementation hints or schema changes.
```
